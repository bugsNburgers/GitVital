## [Unreleased]

### Added
- **`50a63d8` — Issue Metrics & Code Churn Engine:** Added pure metrics modules `backend/src/metrics/issueMetrics.ts` and `backend/src/metrics/churnMetrics.ts` to compute open issue backlog indicators (`openIssueCount`, `avgIssueAgeDays`, `unrespondedIssuePct`) and repo-level weekly churn (`avgWeeklyChurn`, `totalChurn`, `churnScore`) directly from paginated GitHub datasets.
- **`c6cb2bf` — Deterministic Risk Flags Generator:** Added `backend/src/metrics/riskFlags.ts` implementing the full threshold ruleset for danger/warning/success/info signals (contributor concentration, PR slowdown, activity decline, issue neglect, positive health signals) and wired output merging behavior into the analysis pipeline.
- **`ffa2ba5` — Metrics/GitHub Structure Completion:** Added the missing architecture-map modules `backend/src/metrics/timeline.ts`, `backend/src/github/queries.ts`, and `backend/src/github/fetchMetadata.ts`, and introduced `TimelineEntry` in shared typing to complete the required file-structure contract.
- **`257803d`, `3fb33ce`, `c7d26bc` — Architectural Master Plan:** Logged the complete technical blueprint (`Planscribble.md`, `Guidesrc.md`). Detailed the algorithms for computing repository health (Bus Factor, PR Turnaround, Commit Velocity decay) and defined the core backend infrastructure requirements (BullMQ, Redis, PostgreSQL) to ensure strict adherence to API rate limits.
- **`8ad58c8` — Core Documentation:** Added the primary `README.md` to establish project identity, outlining the goals of GitVital (formerly RepoPulse) and the specific constraints of operating within the GitHub GraphQL free-tier API budget.
- **`ff50536` — Monorepo Foundation:** Initialized the dual-stack architecture. Created the `/backend` directory populated with Node.js, Express, and standard `tsconfig.json` configurations. Created the `/frontend` directory via `create-next-app` utilizing Next.js 14, standardizing on Tailwind CSS for utility-first styling and the App Router for modern component rendering.
- **`5e3f51b` — API Server Initialization:** Implemented the core Express application in `/backend/src/server/index.ts`. Secured the API by configuring essential middleware including `helmet` (for HTTP header security), `cors` (for cross-origin resource sharing from the frontend), and `express-session` (for GitHub OAuth state management).
- **`bc3f51c` — BullMQ Background Job Queue:** Designed and integrated the asynchronous queuing system to prevent the main Express server from freezing during intensive GitHub API fetching. Configured worker threads to process repository analysis jobs asynchronously using `ioredis` to manage task states.
- **`b18e3e8` — Dynamic Routing Next.js Pages:** Scaffolded the initial frontend react components including the static landing page (`/`) and the parameterized repository dashboard route (`/[owner]/[repo]`) to handle client-side navigation.
- **`2ec82fd` — Dedicated Developer Contribution Pipeline:** Engineered the `userAnalyzer.ts` BullMQ worker. This pipeline is exclusively dedicated to computing an individual contributor's open-source footprint, independently executing GraphQL queries against their globally merged PRs across foreign repositories.
- **`ce48363` — Autonomous Cron Scheduling:** Deployed the `node-cron` scheduled execution system. Engineered automation to programmatically trigger repository refresh jobs at 2:00 AM and recalculate developer leaderboard percentile rank standings at 3:00 AM server time, ensuring metrics remain fresh without continuous manual polling.
- **`e0feb53` — Typed GraphQL Client with Rate Limit Armor:** Constructed a custom `GitHubClient` class tailored for GraphQL. Implemented automatic response interception for GitHub's `rateLimit { remaining, resetAt }` headers. Built-in algorithmic exponential backoff (1s, 3s, 9s pauses) to defensively stall requests when hitting `5xx` server limits.
- **`feaf834` — Cursor-Based Data Ingestion Layers:** Built the specialized GraphQL fetching modules (`fetchCommitsTop`, `fetchPullRequests`, `fetchIssues`). Implemented strict mathematical constraints (max 1000 commits, 500 PRs) and infinite-loop failsafes within the recursive pagination logic to rigidly cap API point burn per analysis cycle.
- **`12a37a2` — PostgreSQL Relational Schema:** Orchestrated the complete database structure via Prisma. Established relational mappings across `users`, `repos`, `analysis_jobs`, `repo_metrics`, and `health_timeline`. Deployed highly optimized B-Tree indexing strategies and initialized a materialized view dedicated to offloading the heavy computational weight of the global Leaderboard leader generation.

- **`a7f7f90` — Strict Zero-Cost Architectural Adjustments:** Streamlined worker configurations, memory allocation, and API polling limits throughout the stack to guarantee the complete infrastructure can successfully boot and operate entirely within the free-tier parameters of Neon (Postgres), Upstash (Redis), and GitHub's GraphQL 5000 points/hour allowance.
- **`eaeb47d` — Deep Quota Telemetry Tracking:** Enhanced the GraphQL client logging mechanism to continuously trace the exact point-drain rate. Added intelligent sleep duration mathematical calculations that automatically pause the worker queue exactly until the precise Unix timestamp when GitHub resets the hourly quota.

- **`5a64acb` (Issue 13.2) — Enforced 24-Hour Cache Cycle:** Multiplied the default Redis cache internal TTL fallback inside `config/index.ts` from `3600` (1 hour) up to `86400` seconds (24 hours). - **`d3901ca` (Issue 13.5) — Demolished Temporary Data Stubs:** Completed the pipeline transition inside `repoAnalyzer.ts`. Purged all mathematical stubs (originally placed during Prompt 2.3) and physically wired the paginated GraphQL query modules alongside actual OAuth token decryption and dynamic `GitHubClient` instantiating.

### Fixed
- **`147f3ed` (Issue 13.1) — Absolute Path Resolution Crash in TypeScript:** Corrected a catastrophic build blocker located within `tsconfig.json`. Enforced `"baseUrl": "./src"` into the `compilerOptions`, resolving severe errors where the compiler refused to resolve mapped module import paths (such as `@metrics/*` and `@github/*`), cleanly restoring standard compilation integrity.
- **`913d1e8` (Issue 13.3) — Redis Cache Key Destabilization:** Harmonized a lethal formatting drift involving template literal strings across disparate codebases. Rewrote `metrics:owner/repo` to explicitly map as the colon-separated standard `repo:metrics:{owner}:{repo}` inside both `repoAnalyzer` worker queues and the API read functions, conclusively eliminating a permanent 100% caching miss vulnerability.
- **`0b213e3` (Issue 13.8) — Fatal Undefined Exception in GraphQL Trees:** Hardened the `userAnalyzer.ts` worker execution path against unexpected corrupt data payloads coming from GitHub. Handled the lack of standard non-nullability by placing robust optional chaining (`?.`) directly onto `pr.repository.owner.login` parameters, successfully preventing immediate process `TypeError` crashes whenever fetching pull requests associated with permanently deleted or orphaned remote repositories.

### Security
- **`66e4608` (Issue 13.11) — Mitigated Cross-Site Scripting (XSS) within SVG Endpoints:** Discovered and intercepted an unescaped input validation flaw when outputting dynamic external content via raw XML documents. Coded a robust custom `escapeXml()` utility arrayed within `server/index.ts`. Wrapped all untrusted string interpolations (such as GitHub usernames) to neutralize maliciously crafted HTML entities or `<script>` injections before the browser is allowed to render the badge data over the DOM.

- **`4f5ce2b`, `ea70936`, `dc765c4` Edge Cases & Failure Scenarios:** Substantially hardened the BullMQ worker pipelines against volatile GitHub API edge states. Implemented deep try/catch blocks allowing partial data ingestion (e.g., successful commits but failed PR fetches), zero-commit repository early-exits, and deleted-repository fallback handling for users. Forced strict mathematical overrides (`PR=50`, `Issue=75`) for missing sub-metrics and hard-capped archived repos at a max health score of 30.
